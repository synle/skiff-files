# Changelog

Per-version notes for Skiff Files. The most recent entries appear first inside each phase block. The single source of truth for the current version is `src-tauri/tauri.conf.json` → `version`.

For higher-level context (project goals, phase exit criteria, conventions, footguns), see `CLAUDE.md`.

## Phase 0.2.x — cross-protocol Skiffsync, multi-window, polish

- **0.2.207** — ✅ **Always-on-top window + preview-pane reveal-in-OS**. (1) New `window_set_always_on_top` Tauri command + `Settings.alwaysOnTop` toggle (Settings → Appearance + command palette action). Useful for keeping Skiff visible while drag-dropping files OUT of another app whose window would otherwise cover it. (2) Preview pane header gets a Folder/Reveal icon next to "Open with default app" — routes through `fsRevealInOs`. Hidden for `sftp://` remotes since the local OS can't reveal them.
- **0.2.206** — ✅ **Per-tab custom title + custom file-kind mapping**. (1) Right-click a tab → "Rename tab…" sets a custom label that overrides the auto-derived basename. Persisted via `SavedTab.customLabel`; saving an empty string clears the override. (2) New `Settings.customFileKinds: Record<extension, FileKind>` lets users override the icon kind for arbitrary extensions (e.g. `rs → code`). Settings → Advanced gets an editor with add/remove. Sort-by-kind continues to use the underlying value so the listing doesn't reorder when the user tweaks the map.
- **0.2.205** — ✅ **Three small wins**: (1) StatusBar shows "(N tagged)" when 1+ visible entries carry a Finder-style color tag — quick distribution check without scanning the listing. (2) Settings → Keyboard "Reset all" button wipes `Settings.shortcutOverrides` after confirm. Disabled when no overrides are set; counts active overrides in its label. (3) `Settings.recentPathsMax` (0–50, default 10) makes the recent-paths cap configurable. 0 disables tracking entirely + clears the existing list — privacy-conscious users can opt out without the Sidebar's Recent section staying frozen on stale entries.
- **0.2.204** — ✅ **Six more rebindable shortcuts + cross-platform Cmd/Ctrl unification + drawer state persistence**. (1) Migrated `browser.focusPathBar` (Cmd+L), `browser.bookmarkCurrent` (Cmd+D), `tabs.newTab` (Cmd+T), `tabs.closeTab` (Cmd+W), `tabs.restoreClosedTab` (Cmd+Shift+T), `app.newWindow` (Cmd+N) to the rebindable framework. (2) `keyEventToCombo` now treats `metaKey || ctrlKey` as a single platform-neutral `cmd` modifier; `matchesCombo` normalizes stored combos so a binding configured on macOS as Cmd+K matches a Ctrl+K press on Linux without per-platform forks. (3) `Settings.operationsDrawerExpanded` persists the drawer's expand/collapse — a user who collapses the drawer no longer has it re-expand on every new job.
- **0.2.203** — ✅ **Four more rebindable shortcuts + configurable date format**. (1) Migrated `app.cheatsheet` (Shift+/), `browser.refresh` (Cmd+R), `browser.togglePreview` (Cmd+I), `browser.newFolder` (Cmd+Shift+N) to the rebindable framework. F1 stays a hardcoded alias for the cheatsheet (Windows / web-app convention). F5 likewise stays hardcoded for refresh. (2) New `Settings.dateFormat: "locale" | "iso" | "short" | "relative"` drives the FileList Modified column. Settings → Appearance gets a dropdown. Pure helper `formatMtimeAs` keeps the rendering logic unit-testable.
- **0.2.202** — ✅ **Three follow-ups landed together**: (1) Per-folder tag-filter persistence — `Settings.folderTagFilter` mirrors `folderKindFilter` (LRU 200), so the chip strip survives navigation. Closes 0.2.199 follow-up. (2) Recursive find honors regex + case-sensitive — new `regex` crate; `fs_find` accepts `regex` + `caseSensitive` flags; invalid regex surfaces inline in the StatusBar. Closes 0.2.200 follow-up. (3) Hex preview for binary / unknown kinds — `HexBody` in PreviewPane renders the classic offset / 16-byte / ASCII layout for the first 32 KB. Closes the Phase 6 "text, image, hex preview" spec bullet.
- **0.2.201** — ✅ **Sort by tag color**. New "tag" `SortKey` (added to FileList + Toolbar dropdown). Tagged entries cluster by color in the canonical TAG_COLORS order (red → orange → yellow → green → blue → purple → gray); untagged rows fall to the bottom regardless of direction. Closes the follow-up flagged in 0.2.199.
- **0.2.200** — ✅ **Search: regex + case-sensitive toggles**. Two compact ToggleButtons appear next to the search field once a query is non-empty (`Aa` for case-sensitive, `.*` for regex — VS Code / IntelliJ convention). Apply to the in-pane filter; recursive find continues to use the Rust-side substring match for now. Invalid regex surfaces inline in the StatusBar instead of clearing the listing. Auto-resets on navigation + on query clear so a stale error message doesn't linger.
- **0.2.199** — ✅ **Filter by tag color**. The filter chip row from 0.2.180 grew a second strip of seven tag-colored circles (red / orange / yellow / green / blue / purple / gray); clicking restricts the listing to entries with that tag. Multiple chips OR. The toolbar's filter badge counts kind chips + tag chips together so the user notices either kind of filter is active. Sort-by-tag is the obvious follow-up; ships when the existing FileList sort plumbing grows access to the Settings.fileTags map.
- **0.2.198** — ✅ **Five more command palette actions**: Refresh current folder, New folder, New tab, Restore last closed tab, Open new window. Routed via `skiff:refresh` / `skiff:new-folder` / `skiff:new-tab` / `skiff:restore-closed-tab` window events so the palette doesn't need a direct ref into Browser / BrowserTabs state.
- **0.2.197** — ✅ **Finder-style color tags**. New `Settings.fileTags: Record<path, TagColor>` (red / orange / yellow / green / blue / purple / gray) persists per-path tags, LRU-bounded at 200 entries. EntryContextMenu's "Tag" row renders the seven Finder colors as a horizontal dot strip — click to set, click the unchecked-circle to clear. The active tag is glance-able as an outline ring on the picked dot. FileList rows render an 8 px colored dot to the right of the name when a tag is set; tooltip surfaces the color name. Color tokens centralized in `util/tagColors.ts` so a future filter UI / Sidebar surface can read the same palette.

Each handler now reads its combo through `activeCombo()` so the Settings → Keyboard Edit/Disable/Reset controls light up. Cmd+B sidebar alias kept for VS Code muscle memory.
- **0.2.195** — ✅ **Drag from Skiff to Finder/Explorer/Desktop**. Adds `tauri-plugin-drag` v2 (Apache/MIT) which handles the per-platform native drag-source plumbing (NSDraggingSource on macOS, IDataObject on Windows, GTK drag-and-drop on Linux). Filters out `sftp://` paths (no real OS handle). Capability `drag:default` enables the plugin command. JS bridge in `api/drag.ts` invokes the plugin URL directly so we don't need an npm wrapper.
- **0.2.194** — ✅ **Keyboard rebinding framework + Settings UI**. New `util/keybindings.ts` (`keyEventToCombo` / `matchesCombo` / `formatCombo` / `activeCombo`) with full unit-test coverage. `Settings.shortcutOverrides: Record<actionId, string | null>` persists rebinds; `null` = user-disabled.
