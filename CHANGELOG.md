Per-version notes for Skiff Files. The most recent entries appear first inside each phase block. The single source of truth for the current version is `src-tauri/tauri.conf.json` ‚Üí `version`.

For higher-level context (project goals, phase exit criteria, conventions, footguns), see `CLAUDE.md`.

## Phase 0.2.x ‚Äî cross-protocol Skiffsync, multi-window, polish

- **0.2.246** ‚Äî ‚úÖ **Phase 3a ‚Äî plain FTP backend (suppaftp)**. New `src-tauri/src/fs/ftp.rs` (~250 lines) ships an FtpClient mirroring the SFTP backend's shape: connect (10s timeout, passive mode default), `list_dir`, `stat`, `read_text`, `read_base64`. Auth supports anonymous (defaults `"anonymous"` / `"anonymous@"`) and user/password. Threading mirrors `SftpClient` ‚Äî single `tokio::sync::Mutex<FtpStream>` plus `tokio::task::block_in_place` for the synchronous suppaftp call sites. `Connection::Ftp` joins the registry enum; `Registry::get_ftp` is the typed lookup helper. `conn_create_ftp` is the new Tauri command; existing `conn_list_dir` / `conn_stat` / `conn_read_text` / `conn_read_base64` now dispatch on the connection kind so `ftp://` URLs flow through the same listing / preview code paths as `sftp://`. New `ftp://<id>/<path>` scheme in `util/location.ts` + `formatFtp` helper. ConnectionsPage gains a "FTP (plain)" protocol option with host / port / user / password fields defaulting to anonymous. Write operations (`mkdir` / `rename` / `remove`), FTPS, and the docker-based integration tests are scoped for Phase 3b ‚Äî `Registry::get_sftp` errors cleanly when those SFTP-only commands are called against an FTP id, so the surface fails fast rather than silently. Rust tests cover the path-join helper + default-auth constants; frontend `location.test.ts` covers the `ftp://` parse / format / round-trip / isRemote paths.
- **0.2.245** ‚Äî ‚úÖ **SQLite-backed thumbnail cache**. New `src-tauri/src/fs/thumbnail.rs` opens `<app_data_dir>/thumbnails.db` at app start (failure is non-fatal ‚Äî falls back to the kind-icon UX). Three new Tauri commands: `fs_thumbnail(path, sizePx)` (returns base64 PNG, hits cache or decode‚Üíresize‚Üíencode‚Üícache on miss), `fs_thumbnail_stats` (count + bytes for the Settings UI), `fs_thumbnail_clear` (DELETE + VACUUM, returns rows deleted). Cache key is `(path, mtime, size, sizePx)` so an edit auto-invalidates and Tile + Gallery views with different thumbnail dimensions coexist without thrashing. `GalleryThumb` (used by Tile + Gallery views) now routes through `fs_thumbnail` instead of `fs_read_base64`, so a 100-image folder loads ~50KB-per-thumbnail instead of full file bytes ‚Äî roughly a 100√ó memory drop on a folder of 5 MB photos. The in-memory LRU on top of it is also now keyed by `(path, size)` so Tile + Gallery don't collide. Settings ‚Üí Advanced grows a "Thumbnail cache" row with a live `count + size` readout and a "Clear thumbnail cache" button. Rust tests cover cache round-trip + invalidation + stats + clear, plus a real PNG decode + resize verifying aspect-ratio preservation. Frontend test pins the new `fs_thumbnail` command call.
- **0.2.244** ‚Äî ‚úÖ **Accessibility audit + top-tier fixes**. New `A11Y.md` captures the audit baseline. Inline fixes: (1) FileList rows now follow the **roving-tabindex pattern** ‚Äî `tabIndex={isFocused ? 0 : -1}` so the focused row is the only Tab stop and keyboard-only users can finally reach the list (was the highest-impact gap); (2) StatusBar selection summary gains `aria-live="polite"`, error bar gains `aria-live="assertive"` so screen readers announce changes; (3) Sidebar host status dot gets `aria-label="Connection status: connected"` (Tooltip alone wasn't propagating to the icon); (4) CommandPalette wired to the standard combobox pattern ‚Äî `role="combobox"` + `aria-expanded` + `aria-controls` + `aria-autocomplete="list"` + `aria-activedescendant` on the input, `role="listbox"` on the list, `role="option"` + `aria-selected` + stable id per option, so screen readers announce the highlighted command as the user types. Lower-priority items (FileList `aria-colcount`, dialog `aria-labelledby`, rename-input focus return, PathBar Tab-hint, theme contrast spot-check) deferred + cataloged in `A11Y.md`.
- **0.2.243** ‚Äî ‚úÖ **i18n scaffold (English bundle)**. New `src/i18n/` module wires up `react-i18next` with a single English translation bundle (`en.ts`) covering the sidebar / toolbar / common-buttons / settings-headings surfaces. `initI18n(language)` is called once before React mounts in `main.tsx`, seeding the locale from the persisted `Settings.language` (default `"en"`) so first paint never flashes English over a non-English preference. Settings ‚Üí Appearance gains a Language dropdown wired to `Settings.language`; switching it calls `i18n.changeLanguage()` live (no relaunch). Sidebar's nine section labels + the Transfers / Settings nav links go through `t(...)` as the first-pass wrap; the rest of the app stays unwrapped at this scaffolding stage and grows over future PRs. Fallback locale = English so a stale settings.json from a future-shipped locale doesn't render keys verbatim. Tests cover the English bundle, fallback behavior, and registered-locale list.
- **0.2.242** ‚Äî ‚úÖ **Image rotate ‚Äî save to disk**. PreviewPane already had Rotate-left / Rotate-right buttons that updated CSS transform locally; this slice adds a third Save button next to them. Save dispatches a new `fs_image_rotate(path, degrees)` Tauri command that decodes the file via the `image` crate (added with default-features=false + an explicit JPEG / PNG / GIF / WebP / BMP feature list to keep bundle weight tight), rotates the pixel buffer in 90¬∞ increments, and writes back atomically (temp + rename) so a crash mid-encode doesn't truncate the original. After save the CSS rotation resets to 0 and the preview re-fetches the now-rotated bytes via a `reloadKey` bump. Lossy JPEG round-trip is documented in the tooltip; PNG / GIF / BMP / lossless-WebP are bit-perfect. Rust tests cover lossless PNG rotation (dimensions swap, color preserved), the no-op fast paths (0¬∞ / 360¬∞ / -720¬∞ don't rewrite the file), and rejection of non-quarter rotations. Closes the long-deferred backlog item.
- **0.2.241** ‚Äî ‚úÖ **Opt-in local crash reporting**. New `Settings.crashReportsEnabled` (default false). When toggled on, a Rust-side `std::panic::set_hook` writes a single timestamped log to `<app_data_dir>/crashes/<ts>_<thread>.log` for every panic ‚Äî capturing the panic message, location, thread, and app version. Local-only, never submitted anywhere. Implementation: new `crash.rs` module + reads the flag straight from `settings.json` in the `tauri::Builder::setup` callback before any Rust code can panic (the frontend hasn't mounted yet at that point, so IPC isn't an option). Settings ‚Üí Advanced gains a "Save crash logs (opt-in, local only)" toggle, helper text noting that flips take effect on the next launch, a live "N logs saved" counter, and a "Reveal crash logs folder" button. Two Tauri commands (`crash_logs_dir`, `crash_logs_count`) + frontend `crashLogsDir()` / `crashLogsCount()` wrappers. Rust-side tests cover the JSON-flag scrape (no panic on missing file / malformed JSON / wrong-type field) and the filename sanitizer.
