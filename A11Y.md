# Accessibility Audit — 0.2.244

Snapshot of the keyboard-only / screen-reader audit done at 0.2.244, the inline fixes that landed in the same release, and the lower-priority items deferred for follow-up.

The audit covered the most-touched surfaces: `FileList`, `Toolbar`, `PathBar`, `Sidebar`, dialogs, `CommandPalette`, `PreviewPane`, and `StatusBar`.

---

## Fixed in 0.2.244

| File | Issue | Fix |
|---|---|---|
| `FileList.tsx` row | Rows were not in the keyboard tab order — `focusedIdx` drove a visual ring but Tab couldn't reach a row. Keyboard-only users were locked out of the file list. | Added `tabIndex={isFocused ? 0 : -1}` (roving tabindex). The focused row is the only Tab stop; Arrow keys shift `focusedIdx` and the next render promotes the new row. |
| `StatusBar.tsx` (selection summary) | `role="status"` without `aria-live` — selection-count + free-space changes weren't announced. | Added `aria-live="polite"`. |
| `StatusBar.tsx` (error bar) | Same gap, but errors are higher urgency. | Added `aria-live="assertive"`. |
| `Sidebar.tsx` host status dot | `<CircleIcon>` inside a `<Tooltip title="Connected">` — Tooltip doesn't propagate `aria-label` to the icon, so screen readers heard "graphic" with no context. | Added `aria-label="Connection status: connected"` directly on the icon. |
| `CommandPalette.tsx` | Generic `<List>` of `<ListItemButton>` with no combobox semantics — screen readers couldn't announce the currently-highlighted command as the user typed. | Wired the standard combobox pattern: `role="combobox"` + `aria-expanded` + `aria-controls` + `aria-autocomplete="list"` + `aria-activedescendant` on the input; `role="listbox"` on the list; `role="option"` + `aria-selected` + a stable `id` on each option. |

## Deferred (still open, scoped + skipped)

Each was scoped during the audit and intentionally deferred — work them on explicit ask.

- **`FileList.tsx` `aria-colcount`** — `role="grid"` carries `aria-rowcount` but not `aria-colcount`. Column count varies with the user's `hideColumns` set (checkbox + name always; size / modified / kind conditional). Mechanical to add but needs the right derivation site.

- **Dialog `aria-labelledby`** — Most dialogs (`ConfirmDialog`, `RenameDialog`, `NewEntryDialog`, `PropertiesDialog`, `KeyRecorderDialog`) rely on MUI's auto-wiring of Dialog ↔ DialogTitle. Explicit `aria-labelledby` is a defensive belt-and-braces. MUI is reliable enough that this hasn't bitten yet.

- **`FileList.tsx` rename input focus return** — Inline rename auto-focuses on mount, but on Esc / blur the focus doesn't return to the row. Keyboard users lose their place. Needs threading a `rowRef` (or focused-path state) into `RenameInput`'s `onCancel`.

- **`PathBar.tsx` Tab-completion hint** — Input is `aria-label="Path"` but the Tab-autocomplete affordance isn't documented in an `aria-describedby`. Low-impact polish.

- **`Toolbar.tsx` search-history datalist** — HTML5 `<datalist>` works for sighted users but the accessibility tree is incomplete (`aria-owns` / explicit combobox would tighten it).

- **Theme contrast spot-check** — Light theme's `primary` (`#1565c0`) on white paper is theoretically AA, but a one-off contrast measurement against actual rendered buttons / links would confirm.

## Reproducing the audit

There's no automated `axe-core` integration in CI yet (deferred). Manual passes:

- **Keyboard-only walk**: `Tab` through every surface; can you reach every action? Does focus return predictably after closing a modal? Does Esc do what the user expects (close the topmost layer)?
- **VoiceOver (macOS) walk**: open the app, hit `VO+→` repeatedly through the main flows — Browser navigation, sidebar, settings. Note any "graphic" / "button" announcements with no name.
- **Reduced motion**: System Preferences → Accessibility → Display → Reduce motion = on, plus Settings → Appearance → Reduce motion = on. The MUI theme already drops transitions when reduced motion is active.

### Quick test for the new wins

1. **FileList row tabIndex**: launch app, click into the file list, press Tab — focus should move TO the file list, then a second Tab should leave it (single tab stop on the focused row). Arrow keys move within.
2. **Command Palette combobox**: open with `Cmd+Shift+P`, type — VoiceOver should announce "Command palette, combobox, expanded, <highlighted command name>" as you arrow.
3. **StatusBar live region**: select files in the Browser — VoiceOver should announce "<N> selected, <bytes>".
4. **Sidebar host dot**: with VoiceOver focused on a connected host, you should hear "Connection status: connected".
